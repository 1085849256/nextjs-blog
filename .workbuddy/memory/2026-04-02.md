## 管理后台安全加固

用户指出直接输入 `/admin` URL 能进入后台存在安全风险。全面加固：

### 安全措施:
1. **创建 `src/middleware.ts`** - Next.js 中间件，三层保护：
   - `/admin/*` 页面路由：无 session token → 302 重定向到 `/login`
   - `/api/posts/[id]` PATCH/DELETE：无 session → 401
   - `/api/likes`、`/api/comments`：IP 级限流（防刷）
   - 全局安全响应头（X-Content-Type-Options, X-Frame-Options 等）

2. **重构 `admin/layout.tsx`** - 从 `'use client'` 改为服务端组件：
   - 使用 `getServerSession(authOptions)` 在服务端验证
   - 未登录 → `redirect('/login')`（302）
   - 非 ADMIN/EDITOR 角色 → `redirect('/')`（无权限）

3. **创建 `src/lib/api-auth.ts`** - API 统一鉴权工具：
   - `requireAuth()` - 检查是否登录
   - `requireAdmin()` - 检查是否为管理员

4. **保护 API 接口**：
   - `/api/posts/[id]` PATCH/DELETE 需要 `requireAdmin`
   - `/api/posts` 列表接口：非管理员查 `status=DRAFT` 会静默降级为 `PUBLISHED`

5. **修复 ThemeToggle 水合错误** - 添加 `mounted` 状态延迟客户端主题渲染
